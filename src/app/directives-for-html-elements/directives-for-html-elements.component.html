Is the component you are creating better described as a "styled html element"? If so, It should be a directive. Here's why:

If you wrap a native html element like a button into a component, then to get the bare minimum feature parity you need to add most attributes for that element as inputs to the component and then pipe them into the element.

Do you really want to be implementing popovertarget into your buttons?

Instead write a directive and apply it to the desired element to style, animate or add behavior. Daisy-ui-style classes are also an option here.
